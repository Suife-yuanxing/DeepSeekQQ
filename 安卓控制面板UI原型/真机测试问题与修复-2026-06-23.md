## 二、修复详情

### 修复 A：`config.js` 直接检测 `NativeBridge`（解决问题 2 的核心 bug）

**文件**：`shared/config.js`

**根因**：原 `config.js` 用 `if (window.NativeApp && window.NativeApp.isNative)` 判断是否在 APK 内。但 `window.NativeApp` 由 `native.js` 定义，而 **12 个页面未加载 `native.js`**（见下表），导致 `NativeApp` 不存在 → `config.js` 走 `else` 分支设 `server_base = location.origin` = `file://` → `api.js` 的 `_base()` 返回 `file://` → 所有 API 请求打到 `file:///api/v1/...` → `Fetch API cannot load file:///api/v1/bots. URL scheme "file" is not supported.`

**未加载 `native.js` 的 12 个页面**：`我的Bot` / `Bot设置` / `API Key管理` / `QQ通道` / `微信通道` / `修改密码` / `数据权限` / `数据面板` / `管理员面板` / `编辑个人资料` / `通知` / `黑名单`

**修复**：`config.js` 直接检测原生注入的 `window.NativeBridge`（`addJavascriptInterface` 注入，一定先于任何 `<script>` 存在），不再依赖 `native.js` 的执行时序：

```js
var _isNative = false;
try {
  if (window.NativeBridge && typeof window.NativeBridge.isNative === 'function') {
    _isNative = window.NativeBridge.isNative();
  }
} catch (e) { _isNative = false; }
if (_isNative || (window.NativeApp && window.NativeApp.isNative)) {
  window.APP_CONFIG.server_base = 'http://129.211.7.67:8766';
} else {
  window.APP_CONFIG.server_base = location.origin;
}
```

**验证**（logcat DIAG 实测）：
- 修复前 `我的Bot.html`：`APP_CONFIG=file://` + `Fetch API cannot load file:///api/v1/bots`
- 修复后 `我的Bot.html`：`APP_CONFIG=http://129.211.7.67:8766` + `HTTP error 401 — http://129.211.7.67:8766/api/v1/bots`（请求正确打到后端，仅 mock token 被拒）

> 注：`NativeApp=undefined` 在这些页面仍存在（因没加载 native.js），但**不影响功能**——`config.js` 现在直接检测 `NativeBridge`。后续可补齐 12 页的 `native.js` 引用以求彻底，但非阻塞。

### 修复 B：返回键改用 `OnBackPressedDispatcher`（解决问题 3）

**文件**：`agnescord-android/android/app/src/main/java/com/deepseekqq/agnescord/MainActivity.java`

**根因**：手机是 Android 16 (SDK 36)，旧 `onBackPressed()` 已废弃；且手势返回双触发会直接走到 `handleBackAtBoundary()` → `finishAndRemoveTask()` → 应用直接退出。

**修复**：改用现代 `OnBackPressedDispatcher` + `OnBackPressedCallback`：
- 有历史 → `webView.goBack()`
- **根页面 → `moveTaskToBack(true)` 退到后台，不杀进程**（解决"直接退出"）

```java
private void setupBackHandling() {
    getOnBackPressedDispatcher().addCallback(this, new OnBackPressedCallback(true) {
        @Override
        public void handleOnBackPressed() {
            if (webView != null && webView.canGoBack()) {
                webView.goBack();
            } else {
                moveTaskToBack(true);  // 根页面退后台而非退出
            }
        }
    });
}
```

同步清理：移除已废弃的 `onBackPressed()`、`handleBackAtBoundary()`、`doubleBackToExit` 字段、`Handler`/`Looper` 导入；`NativeBridge.onBackPressed()` 改为委托 `getOnBackPressedDispatcher().onBackPressed()`。

### 修复 C：统一页面顶部色块渐变（解决问题 4）

**文件**：`shared/tokens.css` + `shared/base.css` + `Bot创建向导.html` + `聊天页.html`

**根因**：各页面硬编码不同 header 渐变——
- `base.css .title-bar`：`linear-gradient(135deg, var(--color-primary), var(--color-primary-light))`（粉）
- `Bot创建向导.html .header`：`linear-gradient(135deg, #F472B6 0%, #F9A8D4 50%, #FFB0A0 100%)`（粉→珊瑚，三色）
- `聊天页.html .title-bar`：`linear-gradient(135deg, var(--color-primary) 0%, var(--color-primary-light) 100%)`（token 双色）
- `登录页.html .header`：无背景色（透明，靠 body 渐变）

切换页面时顶部色块跳变 = "不同步"。

**修复**：
1. `tokens.css` 新增统一变量 `--gradient-header: linear-gradient(135deg, #F472B6 0%, #F9A8D4 100%)`
2. `base.css .title-bar` 改用 `var(--gradient-header, ...)`（所有继承页自动统一）
3. `Bot创建向导.html`、`聊天页.html` 的自定义 header 改用同一变量
4. `登录页.html` 保留浅色设计（它本就该与功能页不同色，属合理设计）

**验证**（视觉模型实测）：首页顶部 = **dark pink**，统一生效。

### 修复 D：MainActivity 诊断加固 + 白屏兜底（解决问题 5）

**文件**：`agnescord-android/android/app/src/main/java/com/deepseekqq/agnescord/MainActivity.java`

**根因**：① `onReceivedError` 只对 `index.html` 失败才显示兜底，`启动页.html` 失败时静默白屏；② 无 `onReceivedHttpError`（旧回调不覆盖 401/404）；③ 无 WebView 远程调试，白屏时无法定位 JS 错误。

**修复**：
1. `WebView.setWebContentsDebuggingEnabled(true)` —— 开启后可用 `chrome://inspect` + CDP 远程调试
2. `onPageFinished` 打印 URL + `[DIAG]` JS 状态（`API`/`NativeApp`/`APP_CONFIG`/`bodyHas`）
3. 新增 `onReceivedHttpError` —— 捕获 HTTP 错误（401/404 等）
4. `showLoadError()` 对**所有** HTML 页面失败显示兜底（不再只认 `index.html`），且跳过非 HTML 资源错误（避免单个 css/js 404 导致整页空白）
5. `启动页.html` 的 `doNavigate()` 加 try/catch —— JS 异常时显示"启动失败 + 重试"而非白屏

用途：注入 JS（如 localStorage 写 token）、实时看 Console/Network、导航到指定页面。

1. 服务器 `.api.env` 已补 `DEEPSEEK_API_KEY`/`DEEPSEEK_MODEL`/`DEEPSEEK_BASE_URL`（与 bot 的 `.env` 同值）
2. `sudo systemctl restart deepseek-api` 重启服务，新进程加载含 key 的 env
