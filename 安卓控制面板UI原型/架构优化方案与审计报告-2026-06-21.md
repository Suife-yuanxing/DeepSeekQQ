#### O7：离线消息状态机形式化（0.5d）

当前 5.4.3 离线重连是自然语言描述，建议形式化为确定状态机：

```
消息生命周期状态机：
                    ┌──────────┐
        compose ──► │  draft   │ (本地草稿，未发送)
                    └────┬─────┘
                         │ send()
                    ┌────▼─────┐
                    │ pending   │ (已入 Hive 队列，等待发送)
                    └────┬─────┘
                    ┌────┴─────┐
              WS通？│           │ WS不通
                    ▼           ▼
              ┌─────────┐  ┌──────────┐
              │ sending  │  │ queued    │ (排队中，WS 重连后发送)
              └────┬────┘  └─────┬─────┘
                   │ ACK          │ WS 重连
              ┌────▼─────┐  ┌────▼─────┐
              │  acked   │  │ sending   │
              └──────────┘  └────┬─────┘
                                 │ ACK
                            ┌────▼─────┐
                            │  acked   │
                            └──────────┘

幂等保证：
  服务端: UNIQUE(bot_id, client_id) → 同一条消息发两次只存一次
  客户端: ACK 收到后 Hive.delete(client_id) → 不会重复发送
  
冲突解决（云端 vs 本地）：
  以服务端 server_id 为权威时间源
  客户端 Hive 缓存 = 云端最新 N 条 + 本地 pending 队列
  合并策略: server_id 去重 → 按 server_id DESC 排列
```

#### O8：预定义错误码体系（0.5d）

当前 5.2 API 版本化只提了 `min_app_version`。建议建立完整错误码表：

| HTTP | code | 含义 | 前端处理 |
|------|------|------|---------|
| 200 | `ok` | 成功 | — |
| 400 | `invalid_phone` | 手机号格式错误 | 输入框标红 + 提示 |
| 400 | `weak_password` | 密码强度不足 | 强度条标红 |
| 400 | `bot_name_taken` | Bot 名字已存在 | 输入框标红 |
| 401 | `token_expired` | Access Token 过期 | Dio Interceptor 自动刷新 |
| 401 | `token_invalid` | Token 无效（refresh 也过期） | 跳转登录页 |
| 403 | `insufficient_permission` | 无权限 | Toast 提示 |
| 404 | `bot_not_found` | Bot 不存在或不属于你 | 返回列表页 |
| 429 | `rate_limited` | 请求太频繁 | 显示 Retry-After 倒计时 |
| 429 | `queue_full` | LLM 排队已满 | 提示用自带 Key |
| 500 | `internal_error` | 服务器内部错误 | Toast + 错误 ID（方便排查） |
| 503 | `maintenance` | 服务器维护中 | 显示维护公告 |

**前后端共同约定此表**，Phase 1 后端返回标准格式 `{"code":"...","message":"..."}`, Phase 2 前端按 code 统一处理。
