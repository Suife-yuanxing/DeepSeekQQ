## 第 1 周：P0 架构基础（因果上下文总线）

### 1.1 `CausalContext` — 跨模块共享因果状态 ✅ 核心

**新建文件**：`DeepSeekQQ/plugins/deepseek/causal_context.py`

**设计**：

```
CausalContext 是一个会话级共享状态对象，所有模块通过它感知彼此状态。
一个模块修改状态 → 其他模块自动感知 → 行为产生因果链。

例：activity_sim 设置当前活动=打游戏
    → CausalContext.current_activity = "打游戏"
    → handler 读取到 current_activity → reply_speed *= 0.6（打字慢）
    → night_reminder 读取到 → "打游戏上头中，忘记说晚安"概率+30%
    → promise_tracker 读取到 → 兑现推迟概率+20%
```

**API**：

```python
@dataclass
class CausalContext:
    # 当前活动状态
    current_activity: str = ""           # "打游戏"/"在图书馆"/"上课"
    activity_intensity: float = 0.5      # 活动投入度 0~1（越投入越不回复）
    activity_started_at: float = 0.0     # 活动开始时间戳

    # 身体状态
    energy: float = 1.0                  # 精力 0~1
    hunger: float = 0.0                  # 饥饿 0~1
    tiredness: float = 0.0               # 疲劳 0~1

    # 情绪状态（从 emotion 模块同步）
    current_emotion: str = "平静"
    emotion_intensity: float = 0.0
    emotion_hidden: bool = False         # 是否隐藏情绪

    # 对话状态
    conversation_depth: int = 0          # 对话轮次
    is_ending: bool = False              # 对话是否在收尾
    last_user_reply_gap: float = 0.0     # 上次回复间隔（秒）

    # 因果链记录（最近N条因果事件）
    causal_chain: list = field(default_factory=list)

@dataclass
class CausalEvent:
    timestamp: float
    cause: str        # "activity_sim: 开始打游戏"
    effect: str       # "reply_speed: 降低到0.6"
    module: str       # 触发的模块
```

### 1.2 `CausalChain` — 因果链注册表

**新建文件**：`DeepSeekQQ/plugins/deepseek/causal_chain.py`

定义因果规则 DSL，使因果链可配置、可审查：

```python
CAUSAL_RULES = [
    # (触发条件, 效果函数)
    CausalRule(
        name="打游戏→回复变慢",
        trigger=lambda ctx: ctx.current_activity in ("打游戏", "打LOL", "打原神"),
        effects=[
            ("reply_speed", lambda ctx: max(0.3, ctx.reply_speed * 0.6)),
            ("忘记晚安概率", lambda ctx: 0.3),
            ("承诺兑现推迟", lambda ctx: 0.2),
        ]
    ),
    CausalRule(
        name="生气→忘记说晚安",
        trigger=lambda ctx: ctx.current_emotion == "生气" and ctx.emotion_intensity > 0.5,
        effects=[
            ("忘记晚安概率", lambda ctx: 0.4),
            ("回复变短", lambda ctx: "minimal"),
        ]
    ),
    CausalRule(
        name="极累→所有主动行为概率降低",
        trigger=lambda ctx: ctx.tiredness > 0.8,
        effects=[
            ("主动消息概率", lambda ctx: 0.1),
            ("回复意愿", lambda ctx: 0.3),
        ]
    ),
    # ... 更多规则
]

async def apply_causal_chain(ctx: CausalContext) -> CausalContext:
    """遍历因果规则，更新上下文状态。"""
    for rule in CAUSAL_RULES:
        if rule.trigger(ctx):
            for target, effect_fn in rule.effects:
                # 应用效果 + 记录因果事件
                pass
    return ctx
```

### 2.1 缺席事件生成器 — 「刚才没回是因为…」


### 2.2 情绪累积触发 — 替代关键词匹配

**修改文件**：`emotion_classifier.py`、`context_analyzer.py`

**方案**：不是单条消息触发情绪，而是累积评分

```
累积情绪模型：
1. 每条消息打分（情绪倾向 + 强度）
2. 累积到阈值才切换情绪状态
3. 累积期间保持"正常"表现
4. 延迟反应：事后反刍（隔1-5条消息才表现）
```

**新增**：`emotion_accumulator.py`

```python
@dataclass
class EmotionAccumulator:
    user_id: str
    session_id: str
    buffer: List[EmotionUnit]       # 待处理的情绪单元
    accumulated_valence: float      # 累积效价
    accumulated_arousal: float      # 累积唤醒度
    threshold: float = 3.0          # 触发阈值

    def feed(self, unit: EmotionUnit) -> Optional[EmotionState]:
        """喂入一个情绪单元，达到阈值返回新情绪状态"""
        self.buffer.append(unit)
        # 旧情绪随时间衰减
        self._decay_old_units()
        # 计算累积值
        total = self._compute_accumulated()
        if abs(total) >= self.threshold:
            return self._trigger_emotion()
        return None  # 还没到阈值，隐藏
```

**预估**：3 个工作日 | **风险**：中 | **依赖**：CausalContext

### 4.2 情绪隐藏引擎

**修改文件**：`emotion_deep.py`、`stages/stage_context.py`

**方案**：
- 情绪检测到 ≠ 情绪表达
- 80% 的轻中度情绪波动被隐藏
- 隐藏情绪以「微表达」形式泄露（标点变化、回复变短、用词微妙不同）

```python
async def should_express_emotion(emotion: EmotionState, user_id: str) -> tuple[bool, str]:
    """决定是否表达情绪，以及以什么方式表达。"""
    # 高强度 → 很难隐藏
    if emotion.intensity > 0.8:
        return (True, "explicit")

    # 中强度 → 可能隐藏，可能微表达
    if emotion.intensity > 0.5:
        if random.random() < 0.4:  # 40% 隐藏
            return (False, "hidden")
        if random.random() < 0.5:  # 剩余60%中一半微表达
            return (True, "micro")  # 微表达：标点减少、回复变短
        return (True, "explicit")

    # 低强度 → 大概率隐藏
    if emotion.intensity > 0.2:
        if random.random() < 0.8:
            return (False, "hidden")
        return (True, "micro")

    return (False, "none")
```

```python
    return f"你当前的心情倾向：{'偏正面' if va.valence > 0.2 else '偏负面' if va.valence < -0.2 else '中性'}，" \
           f"精力{'充沛' if va.arousal > 0.6 else '一般' if va.arousal > 0.3 else '低落'}。" \
           f"说话语气自然反映这个状态，但不要直接说'我心情很好/很差'。"
```

### 6.2 情绪残留系统

**修改文件**：`emotion_deep.py`

**方案**：
- 情绪从"恢复→干净"改为"恢复→残留淡出"
- 残留指数：刚恢复时 0.3，随时间指数衰减
- 残留影响：语气比平时淡 10-30%，偶尔突然想起又波动一下

```python
@dataclass
class EmotionResidue:
    emotion_type: str
    residue_intensity: float   # 0~1，初始0.3，指数衰减
    decay_rate: float = 0.1    # 每小时衰减10%
    last_rekindle: float = 0.0 # 上次复发时间

    def current_intensity(self) -> float:
        elapsed_hours = (time.time() - self.last_rekindle) / 3600
        return self.residue_intensity * math.exp(-self.decay_rate * elapsed_hours)
```

**预估**：2 个工作日 | **风险**：低
